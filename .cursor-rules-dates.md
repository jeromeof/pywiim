# Date Format Rules

## CRITICAL: Always Use Actual Dates

When updating CHANGELOG.md or any date fields:

1. **ALWAYS use the actual current date** in YYYY-MM-DD format
2. **NEVER use placeholder dates** like:
   - "2025-01-XX"
   - "2025-XX-XX" 
   - "2025-XX-13"
   - Any date with "XX" or placeholders

3. **How to get the correct date:**
   ```bash
   date +%Y-%m-%d
   ```
   Use the EXACT output from this command.

4. **Check existing format:**
   - Look at previous changelog entries
   - Format is always: `## [VERSION] - YYYY-MM-DD`
   - Example: `## [1.0.7] - 2025-11-13`

5. **If unsure, check the system date first:**
   ```bash
   date +%Y-%m-%d
   ```

## Why This Matters

- Placeholder dates look unprofessional
- They require manual fixes
- They break automation
- They confuse users reading the changelog

## Remember

**NO PLACEHOLDERS. EVER. ALWAYS USE THE ACTUAL DATE.**
